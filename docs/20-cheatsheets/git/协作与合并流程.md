## 结论
团队统一 `git pull --no-rebase`，按最小改动集提交，冲突解决后立即提交。

## 日常同分支协作
```bash
# 从远端拉取并合并到当前分支（merge 风格，避免 rebase 改写历史）
git pull --no-rebase origin <branch>
# 修改代码
# 暂存本次改动（建议按功能点选择文件）
git add <files>
# 生成一次可追溯提交
git commit -m "<message>"
# 推送当前分支到远端
git push origin <branch>
```

## 分支合并（test -> main）
```bash
# 切换到目标分支 main
git switch main
# 拉取远端 main，确保在最新基础上合并
git pull --no-rebase origin main
# 将 test 分支合并进 main
git merge test
# 解决冲突
# 冲突解决后重新暂存并提交合并结果
git add <files>
git commit -m "merge test into main"
# 推送 main
git push origin main
```

## 远端变更只看不合并
```bash
# 更新远端引用，不改工作区
git fetch origin
# 查看本地 HEAD 到远端分支之间的提交差异
git log --oneline --decorate HEAD..origin/<branch>
```

## 注意事项
- 推送被拒先 `pull --no-rebase` 再解冲突。
- 避免一次 commit 混入多个问题域。
