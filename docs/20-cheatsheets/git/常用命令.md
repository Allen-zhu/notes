## 结论
优先记录可直接复制执行、且高频稳定的命令。

## 操作
- `git status`：查看工作区、暂存区和分支状态；每次提交前先执行，确认改动范围。
- `git log --oneline --decorate --graph -20`：以紧凑图形化方式查看最近 20 条提交；用于快速理解分支历史和合并关系。
- `git diff`：查看未暂存改动内容；若要看已暂存改动，使用 `git diff --staged`。
